### 2026-03-07 — Engine model inheritance edge case

Fixed config resolution so globals' `engine.model` is only inherited when the project uses the same engine as globals. Previously, a project with a different engine (e.g. copilot) and no `engine.model` would get globals' model (e.g. opencode/gpt-5-nano), causing the wrong CLI to receive an invalid model. Change in `src/config.ts`: `engineModel` is now set from project's model if present; otherwise from globals' model only when `globals.engine.name === project engine`; else `undefined` so the project's engine uses its own default. Adapters already only pass `--model` when model is truthy, so no engine changes needed.

### 2026-03-07 — Copilot final-answer extraction + format research

Added best-effort `extractFinalAnswer()` in the Copilot adapter to trim reasoning/tool output when the model (e.g. Codex-style) emits a clear lead-in (e.g. "Here are the last 3 commits:"). Researched recurring patterns and expanded lead-in regexes (markdown headers, bold labels, natural-language phrases). Researched which engines support --json/stream-json: Claude and Antigravity already use it; Codex, Cursor, OpenCode support it but we don't use it; Copilot has no such flag. Integrated this into backlog drafts 047–050 (OpenCode, Cursor, Antigravity, Copilot log streaming) with an "Output format support" section per task and recommendations (adopt stream-json where available; Antigravity already has it, wire events to progress; Copilot heuristic-only until CLI adds structured output).

### 2026-03-07 — Copilot/Codex LaTeX formatting (no code change)

User reported "weird" LaTeX-style output from gpt-5.3-codex (raw `\(`, `\frac{}{}` in Telegram). Confirmed our changes did not cause it: we only fix model inheritance and slice the final answer; we don't modify response text. Inconsistency across Copilot models (plain text vs LaTeX) is model-specific behavior; OpenCode appearing "cleaner" is due to different models/CLIs and plain-text defaults. No code change; explained that any fix would be a new feature (e.g. prompt or LaTeX→plain conversion) not a regression fix.
