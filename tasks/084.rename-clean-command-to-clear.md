# 084 — Rename /clean command to /clear

**Status**: in progress

## Problem

HAL currently uses `/clean` to reset a session. Other agentic platforms (e.g., Copilot and Claude Code) use `/clear`, so the command name is inconsistent with user expectations.

## Goal

Make `/clear` the canonical reset command name across product surfaces while removing `/clean` from both command and config surfaces.

## Context & Constraints

- This refinement is for the rename/migration task only.
- Do not include response-template wording standardization work from task `082` beyond what is required to keep docs/config references accurate.
- Telegram command behavior must remain unchanged except for command naming and migration messaging.
- If reset succeeds and any follow-up dispatch fails (relevant to prefill flow), the cleared session remains cleared; no rollback attempt.
- Keep explicit `/clear` command analytics/event visibility (do not collapse into generic text-only analytics).

## What to do

### Command surface

- Introduce `/clear` as the primary and only user-facing reset command in Telegram UX surfaces (menus/help/examples).
- Remove `/clean` from user-facing command routing and documentation.

### Behavior parity

- `/clear` should perform the same session reset logic as `/clean`.
- Update help text, docs, and command lists to reference `/clear`.

### Compatibility & migration

- `/clean` command alias should not remain available.
- Add a migration note in docs/changelog to announce `/clean` removal and `/clear` replacement.
- Rename config key usage from `commands.clean` to `commands.clear`.
- Do not support legacy `commands.clean` config; treat it as an invalid configuration key with a clear startup error.
- Ensure command routing remains correct for Telegram; CLI command-surface changes are out of scope unless code currently exposes `/clean` there.

## Acceptance criteria

- [ ] `/clear` resets the session exactly like `/clean`.
- [ ] `/clean` is no longer available as a command surface entry (no alias handling).
- [ ] All relevant documentation and command menus show `/clear` as the canonical reset command.
- [ ] `commands.clear` is the canonical config key.
- [ ] Legacy `commands.clean` config is rejected with a clear configuration error.
- [ ] A migration note exists (docs/changelog) explaining `/clean` removal and that only `commands.clear` is supported.
- [ ] Explicit `/clear` command analytics/events remain available after the rename.
