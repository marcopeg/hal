# Plan — 062 Wizard env file precedence

## Phase 1 — Implement four-case env file selection

### Step 1.1 — Make `pickEnvPath()` async and implement the four-case matrix

File: `src/wizard/confirm-and-write.ts`

Current `pickEnvPath()` is synchronous and only handles two cases. Replace it with an async function that:

| State on disk | Behavior |
|---|---|
| Neither `.env` nor `.env.local` | Return `.env` (no prompt) |
| Only `.env.local` exists | Prompt: write to `.env.local` / stop |
| Only `.env` exists | Prompt: write to `.env` / create `.env.local` / stop |
| Both exist | Prompt: write to `.env` / write to `.env.local` / stop |

- Use the existing `select` primitive from `@clack/prompts`
- "Stop" option returns a sentinel value (e.g. `null`)
- The existing custom-`env`-key path (top of function) is unchanged — it bypasses selection entirely

### Step 1.2 — Handle the "Stop" return in `runConfirmAndWrite()`

- `pickEnvPath()` now returns `string | null`
- When `null`: skip env file write, log a clear message ("Env file not written. You can set the values manually."), then continue to the "What next?" prompt (or exit cleanly)
- No partial writes — config file is written before the env step, so log that config was written but env was skipped
- Update both call sites of `pickEnvPath()` (preview display at line 33 and write at line 62) to handle `null`

### Step 1.3 — Update preview display

When showing the proposed env entries (lines 32–41), call `pickEnvPath()` once early and store the result. If `null` at that point isn't known yet (selection happens later), show "env file: will be selected" or defer the header line until after selection. Simplest approach: move the `pickEnvPath()` call before the summary display and show the chosen path (or "will prompt") in the preview.
