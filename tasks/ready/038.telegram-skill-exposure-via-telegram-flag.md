# 038 — Telegram skill exposure via telegram flag

## Status

ready

## Summary

When exposing skills to Telegram, include only skills explicitly marked with `telegram: true`. Remove the `public` frontmatter key entirely; `telegram` is the sole criterion for Telegram visibility (slash menu and `/help`).

## Context and constraints

- **Backward compatibility:** None. Only `telegram: true` counts. Skills with `public: true` (or any value) and no `telegram` key are not shown in Telegram; the `public` key is no longer read or used.
- **Single flag:** Remove `public` from the codebase and from all docs/examples. Do not keep `public` for future use.
- **Exposure points:** Today, `/help` (HAL_COMMANDS in `message.ts`) filters skills by `public`; the Telegram command menu (`setMyCommands` in `bot.ts` and `watcher.ts`) currently sends all loaded commands with no skill filtering. Both must use the same rule: include a skill only when `telegram: true`.

## Requirements

- Change Telegram skill exposure filtering to require `telegram: true` (no fallback from `public`).
- Remove parsing and use of `public` from the loader and from `CommandEntry`; parse and use `telegram` instead.
- Update `/help` (HAL_COMMANDS) filtering to use `telegram: true`.
- Filter the command list passed to `setMyCommands` (bot startup and watcher) so that skill entries without `telegram: true` are excluded; non-skill commands (builtin, project, system, git) are unchanged.
- Update all documentation and examples that mention `public` in the skill/Telegram context to use `telegram` instead. Scope:
  - **Example skills:** every SKILL.md in the repo that has frontmatter (e.g. `.agents/skills/`, `examples/obsidian/.agents/skills/` or `.claude/skills/`). Replace `public: true` or `public: false` with `telegram: true` or omit the key (skills hidden by default).
  - **README:** root README and any other README that refer to skill visibility or `public` in this context.
  - **docs/:** every file under `docs/` (including all subfolders) that mention `public` for skills or the HAL_COMMANDS "Project Skills" visibility rule — update to `telegram` and align wording (e.g. "Add `telegram: true` to a skill's frontmatter to include it in the command list and in `/help`").
- Document in implementation/refinement notes that there is no backward-compatibility behavior: `public` is removed; only `telegram: true` exposes a skill to Telegram.

- The Telegram slash command menu (setMyCommands) only includes skills with `telegram: true`; other command types are unchanged.
