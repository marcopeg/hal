  - `ensureUserSetup`
  - empty/whitespace guard
  - session loading via `shouldLoadSessionFromUserDir(...)` + `getSessionId(...)`
  - processing status/progress updates
  - engine execution + parsing
  - session save (`saveSessionId`) when returned

- Update the text handler to call the extracted helper so existing normal message flow stays intact.

## Phase 3: Post-clean dispatch behavior

### 3.1 — Run reset first, then dispatch payload
- In `createCleanHandler(...)`, keep `await resetSession(ctx, gramCtx, { silent: true })` as the first step.
- If reset fails, return the existing error reply and do not dispatch payload.

### 3.2 — Payload path: skip clean confirmation and run normal pipeline
- If payload exists:
  - skip configured `commands.clean.message`
  - call the extracted shared message-dispatch helper with the payload
- This guarantees normal processing/progress UX and engine routing.

### 3.3 — No-payload path: keep current confirmation behavior
- If no payload exists:
  - keep existing template resolution (`commands.clean.message` or default)
  - keep existing confirmation reply
