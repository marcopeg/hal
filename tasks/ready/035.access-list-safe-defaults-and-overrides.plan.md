With full-object replacement logic:
```typescript

  allowedUserIds:
    (project.access !== undefined
      ? project.access.allowedUserIds
      : globals.access?.allowedUserIds) ?? [],
  dangerouslyAllowUnrestrictedAccess:
    (project.access !== undefined
      ? project.access.dangerouslyAllowUnrestrictedAccess
      : globals.access?.dangerouslyAllowUnrestrictedAccess) ?? false,
```

Key semantics:
- If `project.access` is `undefined` (key omitted) ‚Üí fall through to globals.
- If `project.access` is defined (even as `{}`) ‚Üí use only project values, ignoring globals entirely. Missing fields within project `access` resolve to their defaults (`[]` for IDs, `false` for the flag).

## Phase 2: Boot-time access validation

### Step 2.1 ‚Äî Create `validateAccessPolicies` function

**File:** `src/config.ts`

Add a new exported function near `validateProjects` (line 530):

```typescript
export function validateAccessPolicies(projects: ResolvedProjectConfig[]): void {
  const errors: string[] = [];

  for (const project of projects) {
    const { allowedUserIds, dangerouslyAllowUnrestrictedAccess } = project.access;
    const hasUsers = allowedUserIds.length > 0;
    const hasUnsafe = dangerouslyAllowUnrestrictedAccess === true;

    if (!hasUsers && !hasUnsafe) {
      errors.push(
        `Project "${project.slug}": no access policy configured. ` +
        `Set access.allowedUserIds or access.dangerouslyAllowUnrestrictedAccess.`
      );
    }
  }

  if (errors.length > 0) {
    throw new ConfigLoadError(
      `Configuration error: invalid access policy\n${errors.map(e => `  - ${e}`).join("\n")}`
    );
  }
}
```

This validates all projects at once and reports all misconfigured projects in a single error message.

### Step 2.2 ‚Äî Wire validation into boot and hot-reload paths

**File:** `src/cli.ts` (line 242, inside `runBotsForConfig`)

After the existing `validateProjects(resolvedProjects)` call, add:

## Phase 3: Auth middleware update

### Step 3.1 ‚Äî Update `createAuthMiddleware` to use the new access model

**File:** `src/bot/middleware/auth.ts`

Replace the current logic with:

```typescript
export function createAuthMiddleware(ctx: ProjectContext) {
  return async (gramCtx: Context, next: NextFunction): Promise<void> => {
    const { allowedUserIds, dangerouslyAllowUnrestrictedAccess } = ctx.config.access;
    const userId = gramCtx.from?.id;

    if (allowedUserIds.length > 0) {
      if (userId && allowedUserIds.includes(userId)) {
        await next();
        return;
      }
      await gramCtx.reply(
        "Sorry, you are not authorized to use this bot.\n" +
        "Contact the administrator to request access.",
      );
      return;
    }

    if (dangerouslyAllowUnrestrictedAccess) {
      await next();
      return;
    }

    await gramCtx.reply(
      "Sorry, you are not authorized to use this bot.\n" +
      "Contact the administrator to request access.",
    );
  };
}
```

Behavior:
- `allowedUserIds` non-empty ‚Üí only listed users pass (dangerous flag ignored).
- `allowedUserIds` empty + `dangerouslyAllowUnrestrictedAccess: true` ‚Üí everyone passes.
- Neither ‚Üí deny (this case should not occur at runtime due to boot validation, but the middleware is defensive).
