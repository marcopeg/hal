# 059 — CLI env file sourcing

**Status**: draft

## Description
Add support for sourcing an environment file before running an engine CLI command.

This is strictly about the spawned engine child process, not the HAL process environment.

The setting is project-scoped and engine-scoped:
- `projects.<key>.engine.envFile` (optional string path)

Path behavior:
- If `engine.envFile` is relative, resolve it from `project.cwd`.
- If `engine.envFile` is absolute, use it as-is.

Execution behavior:
- The env file must be shell-sourced for the child process that runs the engine CLI command (conceptually: `source <envFile> && <engine-cli> ...`).
- This task does not change top-level config `env` behavior used for config substitution.

## Goals
- Extend project config to support optional `projects.<key>.engine.envFile`.
- Implement deterministic path resolution from `project.cwd` for relative values.
- Ensure env file sourcing applies only to spawned engine CLI processes.
- Keep behavior unchanged when `engine.envFile` is not configured.

## Constraints and Expectations
- Runtime scope: child process only; no mutation of HAL's own `process.env` semantics.
- Validation scope: fail boot only for active project(s) whose `engine.envFile` is invalid.
- Error mode: missing/unreadable configured env file is a boot-time error for the affected active project(s).
- Backward compatibility: projects without `engine.envFile` continue to run exactly as today.

## Acceptance Criteria
- A project configured with `projects.<key>.engine.envFile: .env` resolves to `<project.cwd>/.env`.
- A project configured with `projects.<key>.engine.envFile: /absolute/path/.env` resolves to that exact path.
- For configured projects, engine commands run with shell env sourcing semantics before command execution.
- HAL startup fails when an active project has `engine.envFile` configured and the file is missing/unreadable, with a clear error message.
- Inactive projects with invalid `engine.envFile` do not block boot.
- Projects without `engine.envFile` are unaffected.
