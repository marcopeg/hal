# 062 — Wizard env file precedence when both .env and .env.local exist

**Status:** draft

## Goal

Decide and document the intended precedence for the setup wizard when **both** `.env` and `.env.local` exist in the config directory, then align behavior and docs accordingly.

## Context

The docs currently say the wizard writes to `.env` (or `.env.local` if that already exists). The implementation in `src/wizard/confirm-and-write.ts` prefers `.env` when it exists and only uses `.env.local` when `.env` is missing. When both exist, this can surprise users and makes the docs ambiguous.

## Scope

- Decide the canonical rule for which env file the wizard should update when both are present
- Update docs to state the rule explicitly (setup wizard + config env behavior)
- If needed, adjust `pickEnvPath()` to match the chosen rule
- Add a small note about how this interacts with `env` in config (custom env path)

## Out of scope

- Changing the overall env loading precedence for runtime config
- Modifying other parts of the wizard flow unrelated to env file choice

## Notes / opinion

I lean toward **preferring `.env.local` when both exist**, since it is the common convention for local-only secrets. If we keep the current behavior (prefer `.env`), the docs should say so explicitly and explain why.
