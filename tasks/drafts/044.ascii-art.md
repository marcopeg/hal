# 044 — ASCII art HAL 9000 banner on startup

## Status

draft

## Summary

Display an ASCII art HAL 9000–style banner when the app starts (e.g. on `hal
start`), so the first thing users see is a recognisable “HAL” boot screen
before configuration loading and bot startup messages.

## Context

- The CLI entry is `src/cli.ts`. The `start` command runs `runStart()`, which
  creates a startup logger and immediately logs “Loading configuration”.
- There is no current boot splash or banner; output is log lines only.
- A HAL 9000–themed banner fits the project name and the “personal assistant”
  bot theme.

## Requirements

- Add an ASCII art banner evoking HAL 9000 (e.g. the red eye, “HAL 9000”, or a
  short “I’m HAL”–style line). Keep it compact so it doesn’t dominate the
  terminal.
- Show the banner when the app starts, before “Loading configuration” (or
  before any other startup log). Banner should appear only for the `start`
  command, not for `init` or other subcommands.
- Output the banner in a way that is readable in a normal terminal (e.g.
  stdout; consider whether to use the startup logger or raw `console.log` so
  the art isn’t wrapped in pino format).
- On config hot-reload, do not show the banner again (only on initial process
  start).

## Optional

- Config or env flag to disable the banner (e.g. for CI or scripted runs).
- Slight styling (e.g. red colour for the “eye” if the terminal supports it);
  must still look fine in plain ASCII.

## Out of scope

- Changing the startup logger format for other messages.
- Banners or art in Telegram (e.g. /start response); this is CLI-only.
