# 082 - Standardize /new and /clear responses

**Status**: draft

## Problem

The /new and /clear commands do not consistently provide a standard confirmation message and guidance. When a follow-up message is sent together with the command, the response order can be unclear.

## Goal

Ensure /new and /clear always respond with:
- "new session created"
- the engine/model that is running
- quick links to the most important commands (like /help)
Only after this should any follow-up message sent with the command be processed.

## What to do

- Update the /new and /clear command handlers to return the standard response payload.
- Ensure any follow-up message attached to the command is processed only after the standard response is delivered.
- Keep behavior consistent across engines/adapters.

## Acceptance criteria

- [ ] /new and /clear responses always include the three required lines.
- [ ] When a follow-up message is sent with the command, it is processed only after the standard response is delivered.
- [ ] Behavior is consistent across all supported engines/models.
