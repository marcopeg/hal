# 049 — Antigravity log streaming

## Status

draft

## Summary

Implement streaming log updates for the Google Antigravity engine, sending incremental progress messages to Telegram as the CLI runs rather than waiting for full completion.

## Problem

The Antigravity adapter currently buffers all output until the process exits. Users see no feedback during long-running sessions. Claude Code already streams progress via `onProgress` callbacks — Antigravity needs an equivalent approach.

## Output format support (JSON / stream-json)

- **CLI support:** Gemini CLI (Antigravity) supports `--output-format stream-json`, emitting JSONL events: `init`, `message`, `tool_use`, `tool_result`, `thought`, `error`, `result`.
- **Current use in HAL:** We **already use** `--output-format stream-json`. The Antigravity adapter passes it and parses the stream to extract the final assistant text and (on success) session/stats. We do not currently surface individual event types (e.g. `thought`, `tool_use`, `tool_result`) as progress.
- **Recommendation:** The format that enables event-driven streaming is already in place. This task should focus on wiring those existing events to `onProgress`: e.g. on `thought` or `tool_use`/`tool_result` events, send short progress messages to Telegram (“Thinking…”, “Running tool…”, “Got result”). The final `result` event continues to drive the delivered response. No format switch is needed; the improvement is to use the stream we already have for incremental UX.

## Goals

- Map existing `stream-json` event types (thought, tool_use, tool_result, message chunks) to meaningful Telegram progress messages.
- Determine which events are best suited for user-visible progress (avoid noise; prefer high-level signals).
- Implement `onProgress` callback integration in the Antigravity adapter using the existing JSONL stream.

## Scope

- Use existing `stream-json` output; map event types to progress messages (no format change).
- Prototype mapping of thought/tool_use/tool_result (and optionally message) events to progress text.
- Implement incremental Telegram message updates via the existing `onProgress` mechanism.
- Preserve existing buffered behavior as a fallback.

## Deliverables

- Mapping of existing stream-json event types to progress messages (and any config/throttling decisions).
- Updated `src/engine/adapters/antigravity.ts` with streaming/progress support.
- Test notes covering typical Antigravity runs with progress updates visible in Telegram.

## Acceptance Criteria

- Telegram users see incremental status messages while an Antigravity session is running.
- The final response is still delivered correctly after streaming completes.
- Fallback to buffered mode works when no streaming signals are detected.
