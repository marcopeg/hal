# 049 — Antigravity log streaming

## Status

draft

## Summary

Implement streaming log updates for the Google Antigravity engine, sending incremental progress messages to Telegram as the CLI runs rather than waiting for full completion.

## Problem

The Antigravity adapter currently buffers all output until the process exits. Users see no feedback during long-running sessions. Claude Code already streams progress via `onProgress` callbacks — Antigravity needs an equivalent approach.

## Goals

- Investigate what output Antigravity emits to stdout/stderr while running (structured JSON, plain text, progress events, etc.).
- Determine whether Antigravity provides a streaming or event-driven output mode.
- Identify meaningful log lines or events that can be surfaced as Telegram progress update messages.
- Implement `onProgress` callback integration in the Antigravity adapter.

## Scope

- Research Antigravity CLI output format and any available streaming flags.
- Prototype log-line parsing to extract actionable progress signals.
- Implement incremental Telegram message updates via the existing `onProgress` mechanism.
- Preserve existing buffered behavior as a fallback.

## Deliverables

- Findings on Antigravity CLI output structure during execution.
- Updated `src/engine/adapters/antigravity.ts` with streaming/progress support.
- Test notes covering typical Antigravity runs with progress updates visible in Telegram.

## Acceptance Criteria

- Telegram users see incremental status messages while an Antigravity session is running.
- The final response is still delivered correctly after streaming completes.
- Fallback to buffered mode works when no streaming signals are detected.
