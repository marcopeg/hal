# 045 — Interactive Setup Wizard

**Status:** draft

## Description

Support an `npx ... wiz` command that reads the current configuration, identifies missing mandatory information, and interactively guides the user through setting it up. The wizard is idempotent: if a value is already configured, it skips that question unless `--reset` is passed, in which case it walks through the entire flow and overrides existing values with the new answers.

## Requirements

### Idempotency

- Skip questions whose answers are already present in the config
- `--reset` flag re-asks all questions, overriding current values

### CLI Pre-fill Flags

The wizard accepts flags that pre-fill specific configuration values. When a flag is provided, the corresponding wizard step is skipped entirely (the value is applied silently). This enables concise, shareable one-liners for documentation and help text:

```
npx hal wiz --engine cursor
npx hal wiz --engine cursor --model claude-sonnet-4-5
npx hal wiz --bot-key 123456:ABC-DEF
```

Supported flags:

| Flag | Wizard step skipped |
|------|---------------------|
| `--engine <name>` | Engine / model picker (engine selection) |
| `--model <name>` | Engine / model picker (model selection) |
| `--bot-key <value>` | Telegram bot token |
| `--user-id <value>` | Telegram user ID |
| `--session <mode>` | Session mode |

- Flags are validated the same way wizard inputs are; if a value is invalid the wizard falls back to prompting for that step instead of failing hard
- Pre-filled values are shown in the confirmation summary at the end so the user can review them before writing

### Steps (minimum)

1. **First project `cwd`** — list sub-folders of the current working directory as selectable options; also offer using the current directory itself
2. **Telegram bot token** — prompt for the API key; offer help ("do you need help?"), and if yes, open the GitHub docs page with setup instructions in the browser
3. **Telegram user ID** — same as above, with optional help link
4. **Additional allowed users** — ask if more users should be added; repeat until the user selects "no, I'm done"
5. **Engine / model** — let the user pick the engine; show the model picker only if the engine exposes an explicit list or supports model selection
6. **Session mode** — offer `shared`, `segregated`, or `off` based on engine capabilities; default to `segregated` when the engine supports it
7. **File system permissions** — configure based on engine capabilities

### UX

- Display progress at every step ("Step 3 of 5" or a progress bar)
- Where multiple choices exist, offer an interactive picker (keyboard ↑/↓ to navigate, Enter to select, or type the option number as a shortcut)
- At the end, display the resulting YAML config and ask the user to confirm before writing
- After writing, ask the user whether they want to start the bot or exit the wizard
