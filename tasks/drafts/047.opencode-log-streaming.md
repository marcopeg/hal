# 047 — OpenCode log streaming

## Status

draft

## Summary

Implement streaming log updates for the OpenCode engine, sending incremental progress messages to Telegram as the CLI runs rather than waiting for full completion.

## Problem

The OpenCode adapter currently buffers all output until the process exits. Users see no feedback during long-running sessions. Claude Code already streams progress via `onProgress` callbacks — OpenCode needs an equivalent approach.

## Output format support (JSON / stream-json)

- **CLI support:** OpenCode supports `--print --output-format json` (single final JSON with session metadata, timing, costs) and `--output-format stream-json` (multiple JSON objects: system init, assistant messages, final result). Stream-json requires `--verbose`. See e.g. [opencode run --print --output-format](https://opencode.ai/docs/cli/) and [PR #533](https://github.com/sst/opencode/pull/533).
- **Current use in HAL:** We do **not** use these flags. The OpenCode adapter runs the CLI without `--output-format` and treats stdout as plain text.
- **Recommendation:** Switching to `--output-format stream-json` (with `--print` and `--verbose` as required) would give discrete, parseable events (init, assistant message chunks, result). That would make it straightforward to (1) send incremental progress to Telegram from each event type and (2) extract the final result reliably. Implementing log streaming on top of stream-json is therefore strongly recommended and should simplify this task.

## Goals

- Investigate what output OpenCode emits to stdout/stderr while running (structured JSON, plain text, progress events, etc.).
- Determine whether OpenCode provides a streaming or event-driven output mode (prefer adopting `stream-json` if available — see above).
- Identify meaningful log lines or events that can be surfaced as Telegram progress update messages.
- Implement `onProgress` callback integration in the OpenCode adapter.

## Scope

- Research OpenCode CLI output format and any available streaming flags; prefer adopting `--output-format stream-json` (see Output format support above).
- Prototype event (or log-line) parsing to extract actionable progress signals.
- Implement incremental Telegram message updates via the existing `onProgress` mechanism.
- Preserve existing buffered behavior as a fallback.

## Deliverables

- Findings on OpenCode CLI output structure during execution.
- Updated `src/engine/adapters/opencode.ts` with streaming/progress support.
- Test notes covering typical OpenCode runs with progress updates visible in Telegram.

## Acceptance Criteria

- Telegram users see incremental status messages while an OpenCode session is running.
- The final response is still delivered correctly after streaming completes.
- Fallback to buffered mode works when no streaming signals are detected.
