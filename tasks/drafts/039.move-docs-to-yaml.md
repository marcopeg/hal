# 039 — Move docs to YAML

**Status:** draft

## Summary

Rewrite all documentation config examples from JSON to YAML so that comments can be embedded directly in the examples. Add a note that JSON and JSONC are also supported, and detail which JSONC features are available. Create a dedicated page per config language (YAML, JSON, JSONC) showing a complete 2-project setup example.

## Goals

1. **YAML-first examples** — Convert every config snippet in the docs to YAML, taking advantage of inline comments to annotate options, defaults, and gotchas directly in the examples.
2. **JSON / JSONC support note** — Add a visible note (e.g. in a "Config formats" section or callout) stating that `hal.config.json` and `hal.config.jsonc` are also supported alongside `hal.config.yaml`.
3. **JSONC feature documentation** — Detail exactly which JSONC features are supported (single-line `//` comments, block `/* */` comments, trailing commas, etc.) so users know the boundaries.
4. **Per-language reference pages** — Create one dedicated page per supported config language:
   - **YAML** (`hal.config.yaml`) — full annotated 2-project example
   - **JSON** (`hal.config.json`) — full 2-project example (no comments)
   - **JSONC** (`hal.config.jsonc`) — full 2-project example with comments demonstrating supported features
5. Each reference page should show a **complete, realistic** config for a setup with 2 projects including globals, engine, access, logging, rate-limit, context, and commands sections.

## Out of scope

- Changing the config loader or adding new format support (YAML and JSONC are already implemented).
- Modifying any runtime behavior.
