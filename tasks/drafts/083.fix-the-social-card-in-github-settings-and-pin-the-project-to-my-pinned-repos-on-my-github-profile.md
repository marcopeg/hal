# 083 - Fix the social card in GitHub settings and pin the project to my pinned repos on my GitHub profile

**Status**: draft

## Problem

The repository's social preview card in GitHub settings is incorrect or outdated, and the project is not pinned on the owner's GitHub profile, which reduces visibility.

## Goal

Update the repository's social preview card and ensure the project is pinned on the GitHub profile.

## What to do

- Update the social preview card image in GitHub repository settings, using the correct asset (create or update the asset in the repo if needed).
- Verify the Open Graph preview renders the updated social card for the repository URL.
- Pin the repository to the GitHub profile via the profile UI.

## Instructions (GitHub UI)

### Update the social preview image

1. Go to the repository on GitHub.
2. Click **Settings** (use the dropdown menu if the tab is hidden).
3. In **Social preview**, click **Edit**.
4. Click **Upload an image...** and choose the intended card image.
5. Ensure the image is PNG/JPG/GIF, under 1 MB, and at least 640x320 (1280x640 recommended).
6. Save, then verify the repository link shows the new social preview image.

### Pin the repository to the profile

1. Click your avatar and go to **Your profile**.
2. In the **Popular repositories** or **Pinned** section, click **Customize your pins**.
3. Select the repository (up to six pins total).
4. Click **Save pins**.
