# 073 — HAL native desktop app UI

## Status

draft

## Summary

Turn HAL from a CLI into a native desktop app with a friendly UI, keeping the Node environment (Chromium-based shell) so non-developers can get to a "few clicks to the haha moment" experience. The app should support self-updates, manage multiple HAL configs and projects, and optionally auto-install OpenCode so users can try HAL on free models immediately.

---

## Goals

- Provide a cross-platform desktop UI (Chromium-based) that wraps the existing Node CLI.
- Deliver a non-developer onboarding flow with minimal steps and clear guidance.
- Support multiple HAL configurations/profiles, each with multiple projects.
- Enable self-update capability for the desktop app.
- Offer an auto-install flow for OpenCode to reduce time-to-first-success.

## Scope considerations

- Evaluate Electron (or equivalent Chromium-based stack) for Node runtime compatibility.
- Determine packaging and auto-update strategy (e.g., autoUpdater, signing).
- Define profile storage location and migration strategy from existing CLI config.
- Implement a first-run wizard for config creation and project selection.

## Acceptance criteria

- A desktop app can launch and run HAL using a selected profile and project.
- Users can create, select, and edit multiple profiles with multiple projects each.
- Self-update path is implemented and documented for supported platforms.
- OpenCode can be installed or verified from within the app when missing.
- First-run onboarding completes in a few clicks without developer setup.
