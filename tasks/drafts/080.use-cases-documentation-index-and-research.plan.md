# Plan - 080 Use cases documentation index and research

## Objective

Design and validate a HAL "Use cases" documentation initiative: define the section structure, research real-world demand, and prepare a prioritized execution queue with one follow-up task per use case.

## Phase 1 - Discovery and framing

1. Confirm current docs information architecture and where a "Use cases" section should live.
2. Define the target audience segments (personal productivity, family operations, coding workflows, documentation workflows).
3. Define tutorial format constraints: brief, actionable, reproducible with HAL config references.

## Phase 2 - Use-case research and brainstorming

1. Gather external signals from Reddit and other public communities about common assistant/agent workflows.
2. Extract recurring patterns and map them to HAL capabilities.
3. Build a longlist of candidate use cases with short problem/solution statements.
4. Tag each candidate with category, complexity, prerequisites, and confidence.

## Phase 3 - Draft the use-case index

1. Create the new docs section and index page skeleton.
2. Add the mandatory initial use cases:
   - Personal CRM in Obsidian
   - Food and weight tracker in Obsidian
   - Family economy tracking
   - Coding on the go for existing projects
   - Docs review on the go
3. Add additional researched candidates with short descriptions.
4. Define a standard per-use-case tutorial template:
   - When to use
   - Setup prerequisites
   - Step-by-step flow
   - Example prompts/commands
   - Common pitfalls
   - Next automation ideas

## Phase 4 - Prioritization and slicing

1. Prioritize use cases using a simple matrix (impact, effort, confidence).
2. Select the first publication batch.
3. Split work into one task per use case in BACKLOG draft/ready flow.

## Phase 5 - Review and handoff

1. Review the index content for clarity and realism.
2. Confirm satisfaction with the use-case list before execution split.
3. Hand off the approved queue for incremental tutorial publishing.

## Success metrics

- Use-case index is approved as complete enough to drive execution.
- Each selected use case has a clear tutorial outline and owner task.
- The first publication batch can be executed independently without further restructuring.
