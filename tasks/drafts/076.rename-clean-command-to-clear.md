# 076 — Rename /clean command to /clear

**Status**: draft

## Problem

HAL currently uses `/clean` to reset a session. Other agentic platforms (e.g., Copilot and Claude Code) use `/clear`, so the command name is inconsistent with user expectations.

## Goal

Rename `/clean` to `/clear` while preserving behavior and keeping backward compatibility where appropriate.

## What to do

### Command surface

- Introduce `/clear` as the primary command name in all UX surfaces.
- Decide whether `/clean` remains as an alias (deprecated) or is fully removed.

### Behavior parity

- `/clear` should perform the same session reset logic as `/clean`.
- Update help text, docs, and any command lists to reference `/clear`.

### Compatibility & migration

- If `/clean` stays as an alias, emit a gentle deprecation notice.
- Ensure both Telegram and CLI (if applicable) route correctly.

## Acceptance criteria

- [ ] `/clear` resets the session exactly like `/clean`.
- [ ] All documentation and command menus show `/clear` as the canonical name.
- [ ] If `/clean` is still supported, a deprecation message is shown and behavior matches `/clear`.
