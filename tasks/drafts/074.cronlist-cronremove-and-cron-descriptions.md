# 074 — CronList/CronRemove commands + cron descriptions

## Status

draft

## Summary

Improve the cron system so every cron exposes a human-readable description and add system commands like `CronList` and `CronRemove`. `CronList` must show both project-level and system-level crons, grouping them separately, and distinguishing active (pending) versus expired jobs. Use the description when available, otherwise fall back to the file name. `CronRemove` should show buttons for all crons and remove the selected cron by name and file.

---

## Goals

- Add a `description` string to cron definitions (system and project).
- Implement `CronList` command with grouped output: project crons and system crons.
- Show active (pending) vs expired crons in the listing.
- Implement `CronRemove` command that lists all crons as buttons and removes the selected cron.

## Scope considerations

- Define where `description` lives for `.md` and `.mjs` crons and how it is parsed.
- Specify how to determine **active vs expired** (e.g., `runAt` in the past, `scheduleEnds` in the past, `enabled: false`).
- Ensure `CronRemove` updates the scheduler after deleting the cron file.
- Decide the user-facing labels for cron buttons (description + filename).

## Acceptance criteria

- `CronList` returns a message with two sections: **Project crons** and **System crons**.
- Each cron entry shows its description when present; otherwise the file name.
- Each cron entry includes active/expired status.
- `CronRemove` shows buttons for every cron, including the cron name and file name.
- Selecting a `CronRemove` button deletes the cron file and unschedules it.
