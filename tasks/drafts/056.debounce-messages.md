# 056 — Debounce messages

**Status**: draft

## Description
Right now if I dictate for too long or if I write a long long message, Telegram splits it up into multiple messages of 4000 characters each. At this moment those messages are executed one after the other, meaning that the text of the first message is sent to the engine, it waits for the answer, then the second message and so forth. We need to debounce this so that if I send a long long text and Telegram splits it, the full message is rebuilt in HAL and sent all at once to the engine.

I think we should introduce something like a 250 ms debounce or something like that. Do research when we refine this task about how Telegram behaves with long long text and what kind of debounce time would be acceptable/optimal for this task.
