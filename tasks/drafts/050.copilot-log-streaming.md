# 050 — Copilot log streaming

## Status

draft

## Summary

Implement streaming log updates for the GitHub Copilot engine, sending incremental progress messages to Telegram as the CLI runs rather than waiting for full completion.

## Problem

The Copilot adapter currently buffers all output until the process exits. Users see no feedback during long-running sessions. Claude Code already streams progress via `onProgress` callbacks — Copilot needs an equivalent approach.

## Output format support (JSON / stream-json)

- **CLI support:** GitHub Copilot CLI does **not** currently support `--json` or any structured/streaming output format. A feature request exists (e.g. [github/copilot-cli#52](https://github.com/github/copilot-cli/issues/52)) for JSON/JSONL output similar to Claude Code or Codex; it is not implemented as of now.
- **Current use in HAL:** We consume plain stdout and (optionally) apply a heuristic in `parse()` to extract the “final answer” when the model (e.g. Codex-style) outputs reasoning + commands + answer in one stream. No structured format is available to use.
- **Recommendation:** Log streaming for Copilot cannot rely on a structured event stream. Implementation will need to use heuristics (e.g. detect lines that look like “Running command…”, “Get last N commits”, or other recurring patterns in Copilot’s plain-text output) and send those as progress updates. If Copilot adds `--json` or `stream-json` in the future, we can revisit and implement event-based streaming and more reliable final-answer extraction.

## Goals

- Investigate what output Copilot emits to stdout/stderr while running (plain text, line patterns, etc.).
- Determine whether Copilot provides any streaming or event-like output (currently it does not offer JSON/stream-json).
- Identify meaningful log lines or patterns that can be heuristically surfaced as Telegram progress update messages.
- Implement `onProgress` callback integration in the Copilot adapter (heuristic-based until CLI adds structured output).

## Scope

- Research Copilot CLI output format and any available streaming flags.
- Prototype log-line parsing to extract actionable progress signals.
- Implement incremental Telegram message updates via the existing `onProgress` mechanism.
- Preserve existing buffered behavior as a fallback.

## Deliverables

- Findings on Copilot CLI output structure during execution (and note that no JSON/stream format exists yet; document any useful line patterns).
- Updated `src/engine/adapters/copilot.ts` with streaming/progress support.
- Test notes covering typical Copilot runs with progress updates visible in Telegram.

## Acceptance Criteria

- Telegram users see incremental status messages while a Copilot session is running.
- The final response is still delivered correctly after streaming completes.
- Fallback to buffered mode works when no streaming signals are detected.
