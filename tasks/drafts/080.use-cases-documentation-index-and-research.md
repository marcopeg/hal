# 080 - Use cases documentation index and research

**Status**: draft

**Plan**: [080.use-cases-documentation-index-and-research.plan.md](./080.use-cases-documentation-index-and-research.plan.md)

## Problem

HAL already supports strong real-world workflows, but there is no dedicated "Use cases" documentation section that shows practical, end-to-end tutorials. Without this, new and existing users do not quickly discover what HAL is good for in daily life and work.

## Goal

Create a new docs section for HAL use cases, starting with a curated use-case index and short tutorial-style pages. During task refinement, run broad brainstorming and online research (Reddit and other public sources) to identify high-value, realistic use cases people already perform with chat agents and automation.

## Initial use cases (must be covered)

- Personal CRM in Obsidian
- Food and weight tracker in Obsidian (weight log + food journal)
- Family economy tracking (shared expense capture + categorization)
- Coding on the go attached to an existing coding project
- Docs review on the go (run docs review loop, answer consistency questions progressively)

## Scope

### Documentation architecture

- Add a dedicated docs area for use cases (index + per-use-case tutorial pages).
- Define a reusable tutorial template so every use case follows the same structure.
- Cross-link use-case docs from existing entry points (README/docs navigation where appropriate).

### Refinement and research

- Brainstorm a broad set of candidate HAL use cases before implementation.
- Investigate online communities (including Reddit) for real patterns people use with AI assistants, chat bots, and agentic workflows.
- Distill findings into HAL-relevant scenarios, grouped by category and feasibility.

### Planning forward work

- Convert the approved use-case index into one follow-up task per use case so tutorials can be published incrementally.
- Prioritize by user value, implementation/documentation effort, and ability to demo quickly.

## Deliverables

- A new docs "Use cases" section with an index page.
- A reviewed brainstorm list of candidate use cases (including externally researched ideas).
- Tutorial outlines for the required initial use cases listed above.
- A backlog proposal that splits publication into one task per use case.

## Acceptance criteria

- [ ] New docs section exists for use cases and is reachable from docs navigation.
- [ ] Use-case index includes the required initial use cases from this task.
- [ ] Refinement output includes externally researched ideas (Reddit + other sources) and source links/notes.
- [ ] Candidate use cases are grouped and prioritized with rationale.
- [ ] Follow-up tasks are proposed as one task per use case for incremental publishing.
- [ ] Final index content is reviewed and approved as "satisfactory" before splitting execution tasks.
