# 068 — Voice recognition output stitching + transcript retention

## Status

draft

## Summary

Polish voice recognition output by stitching partial transcripts into a single message, and add a config option that keeps the transcript message permanent in chat.

## Problem

Voice recognition output is fragmented across partial messages, and the transcript message can be ephemeral, which makes the final text hard to read or loses context in chat history.

## Goals

- Stitch partial voice recognition output into a single, polished transcript message.
- Add a config option that preserves the transcript message permanently in the chat.
- Ensure transcript message delivery stays consistent across the voice recognition flow.

## Constraints

- Keep default behavior unchanged unless the new config option is enabled.
- Avoid breaking existing message update/progress behavior.
- Custom Whisper prompt injection is out of scope for this task (track separately if needed).

## Acceptance Criteria

- Voice recognition output is delivered as a single stitched transcript message.
- A config option can keep the transcript message permanent in chat history.
- Default behavior remains unchanged when the option is not set.
