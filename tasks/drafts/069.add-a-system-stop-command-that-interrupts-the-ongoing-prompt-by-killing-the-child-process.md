# 069 — Add system stop command that interrupts the ongoing prompt by killing the child process

## Status

draft

## Summary

Add a system stop command that lets users interrupt an active prompt by terminating the underlying child process.

## Problem

There is no reliable way to stop a long-running prompt once it has started. Users need a clear, system-level command that can immediately interrupt the active child process and return control without waiting for completion.

## Goals

- Provide a system stop command that is discoverable alongside other system commands.
- Terminate the active child process cleanly when a prompt is running.
- Return a clear confirmation message to the user when the stop command succeeds.

## Constraints

- Avoid interrupting idle sessions or unrelated background work.
- Keep existing prompt execution behavior unchanged unless the stop command is invoked.
- Ensure the stop command is safe to call when no prompt is active.

## Acceptance Criteria

- A system stop command is available to users.
- When a prompt is active, invoking the stop command kills the child process and stops the prompt.
- When no prompt is active, the command responds safely without crashing.
