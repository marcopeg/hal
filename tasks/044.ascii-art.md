# 044 — ASCII art HAL 9000 banner on startup

## Status

completed

## Summary

Display an ASCII art HAL 9000–style banner when the app starts (e.g. on `hal
start`), so the first thing users see is a recognisable “HAL” boot screen
before configuration loading and bot startup messages.

Refinement intent: keep the startup experience visually themed, compact, and
portable in terminal output, with no additional startup options at this stage.

## Context

- The CLI entry is `src/cli.ts`. The `start` command runs `runStart()`, which
  creates a startup logger and immediately logs “Loading configuration”.
- There is no current boot splash or banner; output is log lines only.
- A HAL 9000–themed banner fits the project name and the “personal assistant”
  bot theme.
- The README uses an image (`images/hal.jpg`) for branding, but startup output
  is terminal text. A literal image embed is not feasible in plain CLI output;
  the task should target text rendering only.

## Clarified scope (refinement)

- Banner content should use **HAL** as the main label (not HAL 9000 / HULL).
- Preferred visual: a compact ASCII approximation of the HAL eye + `HAL` label.
- Image embedding is out of scope; only text/ANSI terminal output is in scope.
- Disable/opt-out flag is deferred (not required in this task).
- Color should be used where possible for effect (e.g. red accent), while
  preserving readable plain output when color is unsupported.

## Requirements

- Add a compact ASCII banner that evokes the HAL eye and includes `HAL` as the
  visible label.
- Show the banner when the app starts, before “Loading configuration” (or
  before any other startup log). Banner should appear only for the `start`
  command, not for `init` or other subcommands.
- Output the banner in a way that is readable in a normal terminal (e.g.
  stdout; consider whether to use the startup logger or raw `console.log` so
  the art isn’t wrapped in pino format).
- On config hot-reload, do not show the banner again (only on initial process
  start).
- Keep startup output robust across terminals:
  - if ANSI color is available, allow a small red accent for the eye;
  - if not, fall back to plain ASCII without broken escape sequences.

## Optional

- Future config or env flag to disable the banner (e.g. for CI/scripted runs).

## Out of scope

- Changing the startup logger format for other messages.
- Banners or art in Telegram (e.g. /start response); this is CLI-only.
- Rendering bitmap/linked images in the terminal.

## Acceptance criteria

- Running `hal start` (or default start command path) prints a compact HAL-eye
  ASCII banner with `HAL` before any startup log line.
- Running `hal init` does not print the banner.
- During config hot-reload in the same process, the banner is not printed
  again.
- Output remains readable both with and without ANSI color support.
- No new config schema or env variable is required for this task.
