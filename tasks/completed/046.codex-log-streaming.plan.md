## Phase 2: Implement `--json` mode and JSONL progress parsing ✅

4. **Add `--json` and `--color never` flags to adapter args** ✅
   Added both flags after permission/sandbox args and before the resume/cwd/prompt args.

5. **Destructure `onProgress` from options** ✅
   Updated destructuring to include `onProgress` alongside `continueSession` and `sessionId`.

6. **Add throttle state and helper** ✅
   Added `lastProgressTime`, `lastProgressMsg`, `THROTTLE_MS = 3000`, and `maybeProgress(msg)` which guards on both time elapsed (≥3s) and duplicate message detection before calling `onProgress`.

7. **Parse JSONL stdout line-by-line for progress** ✅
   Replaced simple string accumulation with a line buffer that splits on `\n`, parses each line as JSON, and dispatches to `progressFromEvent()` for short message generation. Raw JSONL continues to accumulate in `stdout` for final result extraction.

   | Event `type` | Item `type` | Progress message |
   |---|---|---|
   | `item.started` | `command_execution` | `"Running: <command (60 char max)>"` |
   | `item.started` | `file_change` | `"Updating files..."` |
   | `item.started` | `reasoning` | `"Thinking..."` |
   | `item.started` | `mcp_tool_call` | `"Tool: <tool name>"` |
   | `item.started` | `web_search` | `"Searching: <query (50 char max)>"` |
   | `item.completed` | `agent_message` | `"Responding..."` |
   | `turn.started` | — | `"Codex is reasoning..."` |

8. **Extract final answer from JSONL output** ✅
   On close (code 0), scans accumulated JSONL for the last `item.completed` event with `item.type === "agent_message"` and uses `item.text` as `output`. Falls back to raw `stdout.trim()` if not found.

## Phase 3: Implement elapsed-time fallback ✅

9. **Fire initial progress message at t=0** ✅
   Immediately calls `onProgress("Codex is working...")` after spawning if `onProgress` is defined.

10. **Start elapsed-time interval** ✅
    `setInterval` at 3000ms; skips tick if JSONL has fired within the last 6s. Otherwise emits `"Codex is working on it... {N}s"`.

11. **Clear interval on process close or error** ✅
    `clearInterval(elapsedTimer)` called at the top of both `proc.on("close")` and `proc.on("error")`.
