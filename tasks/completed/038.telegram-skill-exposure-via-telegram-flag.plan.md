# Plan: 038 — Telegram skill exposure via telegram flag

## Phase 1: Loader — replace `public` with `telegram` ✅

**Goal:** Parse `telegram` from SKILL.md frontmatter and expose it on CommandEntry; remove all use of `public`.

1. **Update `CommandEntry`** ✅ in `src/bot/commands/loader.ts`:
   - Replace `public?: boolean` with `telegram?: boolean` (from SKILL.md frontmatter `telegram: true`). Update the comment accordingly.

2. **Update `parseSkillMd`** ✅ in `src/bot/commands/loader.ts`:
   - Replace `publicMatch` with a match for `telegram:\s*(.+)` in the frontmatter.
   - In the returned object, replace `public` with `telegram: telegramMatch?.[1].trim().toLowerCase() === "true"`.
   - Update the function's return type: `public: boolean` → `telegram: boolean`.

3. **Update `scanSkillsDir`** ✅ in `src/bot/commands/loader.ts`:
   - When pushing a skill entry, set `telegram: parsed.telegram` instead of `public: parsed.public`.

4. **Update `resolveSkillEntry`** ✅ in `src/bot/commands/loader.ts`:
   - In the returned `CommandEntry` object, add `telegram: parsed.telegram` so resolved skill entries carry the flag for consistency.

---

## Phase 2: Filter skills for Telegram exposure ✅

**Goal:** Use `telegram: true` as the criterion for including skills in the Telegram command menu and in `/help`. Add a single helper so both call sites share the same logic.

5. **Add `commandsForTelegramMenu`** ✅ in `src/bot/commands/loader.ts`:
   - New exported function: `commandsForTelegramMenu(commands: CommandEntry[]): CommandEntry[]`.
   - Return entries where: `source !== "skill"` OR `entry.telegram === true`. So non-skill commands are always included; skills are included only when `telegram` is true.
   - This list is what gets sent to `setMyCommands` and is the same set of commands that should appear in the help "Project Skills" section.

6. **Update `buildHalCommands`** ✅ in `src/bot/commands/message.ts`:
   - In the `case "skill":` branch, change `if (entry.public)` to `if (entry.telegram)` so the Project Skills section only lists skills with `telegram: true`.

7. **Update bot startup** ✅ in `src/bot.ts`:
   - After `loadCommands(...)`, compute `commandsForMenu = commandsForTelegramMenu(commands)`.
   - Use `commandsForMenu` (not `commands`) for `getCommandsWithDescriptionTooLong` and for `bot.api.setMyCommands(...)`. Use `commandsForMenu.length` in the log so the count reflects what was actually registered.

8. **Update command watcher** ✅ in `src/bot/commands/watcher.ts`:
   - After `loadCommands(...)` in `republish()`, compute `commandsForMenu = commandsForTelegramMenu(commands)`.
   - Use `commandsForMenu` for `getCommandsWithDescriptionTooLong` and for `bot.api.setMyCommands(...)`. Use `commandsForMenu` in the log (count and command names) so hot-reload registers the same filtered set.
