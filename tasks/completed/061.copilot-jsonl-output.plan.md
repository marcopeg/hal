## Phase 1 — Version registry & utility (`src/engine/versions.ts`)

### Step 1.1 — Define `VersionMismatchError`
Create a named error class so callers can distinguish a version failure from a "CLI not found" failure:
```ts
export class VersionMismatchError extends Error {
  constructor(public readonly engineName: string, public readonly installed: string, public readonly minimum: string, public readonly upgradeUrl: string) { ... }
}
```

### Step 1.2 — Define the registry
```ts
interface EngineVersionEntry { min: string; upgradeUrl: string; }
const ENGINE_VERSIONS: Record<string, EngineVersionEntry> = {
  copilot: { min: "0.0.422", upgradeUrl: "https://docs.github.com/en/copilot/how-tos/set-up/install-copilot-cli" },
  claude:  { min: "0.0.0",   upgradeUrl: "https://docs.anthropic.com/en/docs/claude-code" },
  codex:   { min: "0.0.0",   upgradeUrl: "https://github.com/openai/codex" },
  cursor:  { min: "0.0.0",   upgradeUrl: "https://cursor.sh" },
  opencode:    { min: "0.0.0", upgradeUrl: "https://opencode.ai" },
  antigravity: { min: "0.0.0", upgradeUrl: "" },
};
```

### Step 1.3 — `parseVersion(stdout: string): string | null`
Parse semver string from `--version` output. Handles common formats:
- `1.2.3`, `v1.2.3`, `copilot 0.0.422`, `Claude Code 1.2.3`, etc.
Use a regex like `/(\d+\.\d+\.\d+)/` to extract the first matching triplet.

### Step 1.4 — `semverGte(a: string, b: string): boolean`
Split on `.`, compare major/minor/patch numerically. No external deps.

### Step 1.5 — `checkEngineVersion(cmd: string, engineName: string): void`
```ts
export function checkEngineVersion(cmd: string, engineName: string): void {
  const entry = ENGINE_VERSIONS[engineName];
  if (!entry || entry.min === "0.0.0") return; // no minimum — skip
  const raw = execSync(`${cmd} --version`, { stdio: "pipe" }).toString();
  const version = parseVersion(raw);
  if (!version || !semverGte(version, entry.min)) {
    throw new VersionMismatchError(engineName, version ?? "unknown", entry.min, entry.upgradeUrl);
  }
}
```

## Phase 2 — Boot-time error handling (`src/bot.ts`)

### Step 2.1 — Catch `VersionMismatchError` around `engine.check()`
In `startBot()`, wrap the existing `engine.check()` call:
```ts
try {
  engine.check();
} catch (err) {
  if (err instanceof VersionMismatchError) {
    logger.error(
      { engine: err.engineName, installed: err.installed, minimum: err.minimum },
      `Engine version too old. Upgrade instructions: ${err.upgradeUrl}`,
    );
    process.exit(1);
  }
  throw err;
}
```

Terminal log is the authoritative output — this is a boot-time crash for the operator. No Telegram notification needed (no reliable chat target exists at boot time before the bot is initialized).

### Step 3.2 — Replace naive stdout accumulation with JSONL parsing
Modelled on the Claude adapter's stream parsing. Key changes:
- Buffer incomplete lines across `data` chunks (split on `\n`)
- For each complete line: attempt `JSON.parse(line)`
- If `event.type === "assistant.message"`: append `event.data?.content ?? ""` to `assistantParts[]`
- If `event.type === "tool.execution_start"`: call `onProgress(\`Using \${event.data?.toolName}...\`)` (if available)
- All other types and non-JSON lines: `logger.debug(...)`, discard

### Step 3.3 — Fallback: raw stdout if no assistant records found

### Step 3.4 — Remove `extractFinalAnswer()`
Delete the function entirely. Update `parse()` to return `result.output` directly (no post-processing needed — extraction is now done in `execute()`).

### Step 3.5 — Wire `checkEngineVersion` into `check()`
