# Plan for 071 — Enforce project cwd in agent prompts

## Goal

Add a HAL-managed cwd anchor instruction to prompts so every engine is explicitly told to treat the resolved project `cwd` as the working boundary, make that behavior opt-out via `engine.enforceCwd`, extend the same prompt wrapper to markdown cron executions, and document the new behavior consistently.

## Execution Notes

- 2026-03-16: Completed the runtime config work by adding `engine.enforceCwd` to the shared engine schema and exposing it in resolved config as `engineEnforceCwd` to match the repo's flattened resolved-config style.
- 2026-03-16: Reused `formatContextPrompt` as the shared prompt seam instead of introducing another helper. This keeps interactive prompts and markdown cron prompts on one formatting path and avoids prompt-order drift.
- 2026-03-16: Applied the cwd instruction only where HAL already constructs the context wrapper, preserving the existing prompt shape and leaving downloads-path handling at the end of the prompt.
- 2026-03-16: Added focused tests for config inheritance, interactive prompt ordering, and cron prompt formatting, then ran `npm run typecheck && npm test && npm run lint` successfully.
- 2026-03-16: Updated the central config/context docs, the engines index, the Copilot path-boundary doc, and the example config. I kept engine-doc churn narrow to pages that already discuss path or sandbox behavior.

## Risks to watch

- Adding the new flag to raw config but forgetting to expose it in resolved config, which would force runtime code to inspect the wrong shape.
- Implementing cwd injection only in `buildContextualPrompt` and missing markdown cron execution, leaving behavior inconsistent.
- Duplicating prompt assembly logic in multiple places and letting ordering drift between interactive and cron paths.
- Accidentally injecting Telegram message-scoped fields into cron context while trying to unify prompt generation.
- Over-editing engine docs that do not actually discuss path boundaries, creating unnecessary doc churn.
