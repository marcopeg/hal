## Context

- OpenCode was added in task 010. The adapter runs `opencode run [-m model]
  [-c] "<prompt>"` with `OPENCODE_DISABLE_CLAUDE_CODE_PROMPT=true`. When no
  explicit `engine.model` is set, HAL does not pass `-m`, so the OpenCode CLI
  uses its own default (no hardcoded default in HAL).
- The adapter is non-streaming: it buffers stdout and returns it on process
  exit; stderr is logged and surfaced as error text on non-zero exit.
- Free services may have different auth requirements, rate limits, or CLI
  behavior than paid providers; the exact failure mode (empty output, wrong
  content, exit code, auth errors) is to be captured during debugging.

### Config: empty `providers` keys (2026-03-05)

To allow `/engine` with only specific engines (e.g. opencode and codex) and no
model lists, use empty keys in YAML:

```yaml
providers:
  opencode:
  codex:
```

- **Schema:** `providers.<engine>` may be `null` or omitted; both are coerced to
  `[]` so the config loads.
- **availableEngines:** Any engine that has a key under `providers` is included
  (even if the list is empty), so `/engine` shows only those engines.
- **/model:** Enabled when the current engine has ≥2 models in its list, or when
  the engine supports self-discovery (OpenCode, Cursor) and the CLI is available.

See `docs/engines/README.md` (Model list, Behavior of /engine).

**No `providers` key:** When the config has no `providers` section at all, at boot
HAL runs a fast check per engine (default CLI `--version`, 2s timeout) and
builds an in-memory list of available engines. If more than one is available,
`/engine` is enabled and shows only those engines. Implemented in
`getAvailableEnginesFromCli()` in `src/engine/cli-available.ts`, used from
`runBotsForConfig` in `src/cli.ts`.

### Providers configuration

- **Schema:** `providers.<engine>` accepts null/empty; only keys present in the input are kept (record + transform). `providers: null` (YAML `providers:` with no sub-keys) accepted and coerced to `{}`. Invalid values per key coerced to `[]`.
- **Merge:** Local config can only override base provider keys, never add new engine keys (`mergeLocalIntoBase` in `src/config.ts`).
- **Discovery:** CLI discovery only when `providers` key is absent. `providers: {}` or empty → no discovery; `/engine` and `/model` disabled. `modelEnabled` requires `availableEngines.length > 0`.
- **Disabled commands:** When `/engine` or `/model` is disabled, text commands and callbacks (`en:`, `md:`) reply *"This command is disabled."* instead of sending to the LLM (`src/bot.ts`).
- **Validation:** When `providers` defines one or more engines, every project's `engine.name` must be in that list; otherwise boot fails with `ConfigLoadError` (`validateProjects` in `src/config.ts`).
