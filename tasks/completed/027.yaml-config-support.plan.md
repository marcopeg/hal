# 027 — YAML config support — Development Plan

## Phase 1: Format detection infrastructure ✅

Implemented in `src/config.ts`. Added `yaml@2.8.2` dependency. Created `resolveConfigFile()` (scans for `.json`, `.yaml`, `.yml` per slot, errors on conflict) and `parseConfigContent()` (routes to `JSON.parse` or `yaml.parse`). Both are exported for reuse by task 028 and `config-writer.ts`. Refactored `loadMultiConfigInternal` and `loadLocalConfig` to use the new helpers. All error messages now reference the actual detected filename. `mergeLocalIntoBase` accepts dynamic filenames. Typecheck passes.

---

## Phase 2: Config watcher update ✅

Expanded `CONFIG_FILES` in `config-watcher.ts` to include `.yaml` and `.yml` variants for both base and local config slots.

## Phase 3: Config writer update ✅

Rewrote `config-writer.ts` to use `resolveConfigFile` and `parseConfigContent` for reading, and a new `serializeConfig` helper for writing (routes to `JSON.stringify` or `yaml.stringify`). Writes back in the same format as the existing file. If no config exists, the function returns early (no-op).

## Phase 4: CLI update ✅

Updated `--cwd` help text to reference `hal.config.{json,yaml}`. Updated startup log annotation to use regex `/hal\.config\.local\.\w+$/` for format-agnostic local override detection. `init` command kept as JSON-only (out of scope).

## Phase 5: Example YAML config ✅

Created `examples/hal.config.yaml` — YAML equivalent of `examples/hal.config.json` with the same structure and values, plus YAML comments.

## Phase 6: Verification ✅

- `pnpm run typecheck` — passes clean.
- `pnpm run lint` — passes clean (after auto-fix of Biome formatting).
- Conflict detection test: placing both `.json` and `.yaml` in examples/ correctly throws `ConfigLoadError` naming both files.
- YAML parsing test: `parseConfigContent` correctly parses YAML content.
- JSON parsing test: `parseConfigContent` correctly parses JSON content.
- Missing config test: `resolveConfigFile` returns `null` when no file found.
