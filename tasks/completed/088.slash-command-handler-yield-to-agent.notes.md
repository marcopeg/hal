# Notes — 088 Slash command handlers can yield to the agent

## What landed

The feature was implemented with an explicit typed command result contract for matched `.mjs` slash commands:

- `{ type: "assistant", message }`
- `{ type: "agent", message? }`
- `{ type: "void" }`

The main runtime change is in `src/bot/handlers/text.ts`, with result normalization extracted to `src/bot/commands/result.ts`.

## Routing decisions

### Matched `.mjs` commands now own routing

Once HAL resolves a matching project/global `.mjs` command, that command decides what happens next.

- `assistant` replies directly to the user
- `agent` forwards to the engine path
- `void` stops routing completely

This means a matched `.mjs` command that returns `{ type: "agent" }` does **not** fall back into the direct Telegram skill shortcut for the same slash command. It goes straight to the engine path.

### Legacy returns kept for compatibility

The implementation keeps old command behavior working:

- raw `string` → treated as `assistant`, with a warning
- falsy non-object (`undefined`, `null`, `false`, `0`, `""`) → treated as `agent`, with a warning

This was intentional to avoid breaking existing user command files while moving docs/examples to the typed contract.

### Malformed typed objects fail fast

Truth-y invalid returns are treated as command errors rather than being silently coerced.

Examples:

- `{ type: "assistant" }`
- `{ type: "agent", message: "" }`
- `{ type: "void", message: "ignored" }`
- `{ type: "unknown" }`

These now surface through the existing `Command failed: ...` path.

## Docs and examples updated

The typed return contract is now documented in:

- `docs/custom-commands/README.md`
- `docs/skills/README.md`

Examples were updated to use typed returns instead of raw string returns:

- `examples/.hal/commands/context.mjs`
- `examples/.hal/commands/joke.mjs`
- `examples/codex/.hal/commands/status.mjs`

The Codex example now also includes a hybrid command+skill flow:

- `examples/codex/.hal/commands/todo.mjs`
- `examples/codex/.agents/skills/todo/SKILL.md`
- `examples/codex/TODOS.md`

This example demonstrates the intended split:

- `/todo` → deterministic command UI with pagination/buttons
- `/todo ...` → `{ type: "agent" }` handoff to the engine/skill

## Helper skills updated

The Codex example helper skills were refreshed so a HAL project can self-author commands and skills with the current contract:

- `examples/codex/.agents/skills/hal_commands/SKILL.md`
- `examples/codex/.agents/skills/hal_cron/SKILL.md`

Notable updates:

- `hal_commands` now teaches skill vs command vs hybrid selection
- it documents `telegram: true` for Telegram-exposed skills
- it explains Telegram command naming/description limits for exposed skills
- it teaches the typed `.mjs` command return contract
- `hal_cron` now explicitly tells the agent not to misuse crons for on-demand features
