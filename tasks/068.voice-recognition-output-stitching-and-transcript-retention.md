# 068 — Sticky transcript with confirmation flow

## Status

in progress

## Summary

Add a `transcription.sticky` config option that changes voice transcription from a volatile in-place message into a permanent confirmation step: HAL shows the final transcript with **Use it / Cancel** buttons, and only forwards the text to the engine if the user confirms.

## Problem

The current voice flow shows a volatile transcription that disappears once the engine response arrives. There is no way to review the transcript before it is sent, and no permanent record of what was transcribed.

Note: partial-transcript stitching across multiple voice messages was explored in task 070 and found unnecessary — Telegram does not split voice messages.

## Goals

- Add `transcription.sticky: true` config option (globals + project level).
- When sticky is enabled, replace the volatile transcription message with a permanent confirmation message.
- User must confirm ("Use it") before the transcript is forwarded to the engine.
- Cancelling discards the transcript; the confirmation message remains in chat (no engine call).
- Confirmation expires after 60 s or when the user sends a new message.

## Flow (when `transcription.sticky: true`)

1. HAL receives audio.
2. HAL sends a volatile status message: *"Transcribing voice…"*
3. Transcription completes. The volatile message is deleted; a **new permanent message** is sent containing the transcript text and two inline buttons: **Use it** / **Cancel**.
4a. User taps **Use it** → transcript is sent to the engine as the user's prompt (buttons removed from message).
4b. User taps **Cancel** → engine call is aborted; buttons are removed from the message; transcript text remains visible in chat.
5. If no button is tapped within **60 seconds**, or the user sends a new message, the confirmation expires: buttons are removed and `[expired]` is appended to the message.

## Config Schema

```yaml
# globals or per-project (project inherits globals, can override)
transcription:
  sticky: true   # default: true (set to false to disable confirmation)
```

```ts
transcription: z
  .object({
    sticky: z.boolean().optional(),
  })
  .partial()
  .optional()
```

## Constraints

- Default behavior (`sticky: false`) is unchanged — volatile message, no confirmation step.
- Stitching partial transcripts within a single audio message is out of scope (task 070 confirmed it is not needed).
- Custom Whisper prompt injection is out of scope.

## Acceptance Criteria

- `transcription.sticky` is accepted at globals and project level; project value overrides globals.
- When sticky is disabled (`transcription.sticky: false`), voice flow is identical to the current behavior.
- When sticky is enabled:
  - The volatile status message is deleted after transcription.
  - A new permanent message is sent with transcript text + **Use it** / **Cancel** buttons.
  - **Use it** forwards the transcript to the engine and removes the buttons.
  - **Cancel** removes the buttons; transcript message stays; no engine call is made.
  - After 60 s of inactivity, or on a new incoming user message, the confirmation expires: buttons removed, `[expired]` appended to message.
