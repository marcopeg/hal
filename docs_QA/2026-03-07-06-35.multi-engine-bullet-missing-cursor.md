## Issue found

The `**Multi-engine support**` feature bullet listed all engines except Cursor:

> use Claude Code, GitHub Copilot, Codex, OpenCode, or Antigravity per project

Cursor is a fully supported engine documented everywhere else (engines table, engines README, config docs), so omitting it here was misleading.

## Question asked

> Should we add Cursor to that bullet explicitly, or simplify the wording so it doesn't enumerate all engines inline?
>
> Options:
> 1. Add Cursor: *"use Claude Code, GitHub Copilot, Codex, OpenCode, Cursor, or Antigravity per project"*
> 2. Simplify: *"use any supported engine per project — Claude Code, Copilot, Codex, OpenCode, Cursor, or Antigravity"*
> 3. Shorten: *"use any supported engine per project (see [Engines](docs/engines/README.md))"*

## User answer

Option 1 — full comma-separated list with each engine linked to its documentation page.

## Fix applied

Updated `README.md` line 15:

```
- **Multi-engine support** — use [Claude Code](docs/engines/claude/README.md), [GitHub Copilot](docs/engines/copilot/README.md), [Codex](docs/engines/codex/README.md), [OpenCode](docs/engines/opencode/README.md), [Cursor](docs/engines/cursor/README.md), or [Antigravity](docs/engines/antigravity/README.md) per project
```
